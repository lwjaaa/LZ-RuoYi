## 需求描述

### 概述

这是一个全新的项目，请帮我实现一个浏览器插件，要求具有企业级UI界面和科技感风格。**色彩规范：主色调为深空蓝（#0f172a / #1e293b），用于背景、导航栏和主要容器；副色调为电光蓝（#3b82f6 / #60a5fa），用于按钮、链接、高亮和交互元素；辅助色为青绿（#10b981）表示成功状态、琥珀（#f59e0b）表示警告、玫瑰红（#ef4444）表示错误/删除。整体采用深色主题，搭配半透明毛玻璃效果和微渐变，营造现代科技感。**要求从淘宝、天猫和1688的商品详情页上获取商品，然后保存到浏览器缓存，并在独立的标签页面上展示。最后在独立标签页上进行导出或者同步到erp系统。涉及到的接口参考下方接口文档。

#### popup功描述

1. 支持批量获取商品和获取单个商品。
2. 只有当前页面是支持的平台的商品详情页，才可以点击获取当前商品。
3. 已获取的商品或者正在获取的商品，不可以再次获取，获取商品时需要校验唯一性。唯一性是通过标签页的标题来判断的。
4. 当点击获取商品信息，或者批量获取。就通过消息通知，触发目标网页的商品信息获取。需要维护一个任务队列，来记录正在执行的商品信息获取任务。
5. popup生命 周期较短，应避免在组件上存储数据，数据强制从本地获取，确保数据一致性。
6. popup需要展示一个logo，点击logo可以打开独立标签页。logo就是插件名称。

#### 独立标签页功能描述

1. 数据获取到保存到了本地，需要从本地获取到这些商品列表进行展示。具体包含商品名称 ，来源url，商品图片列表，规格选项列表，变体列表。其中，规格选项列表包含规格名称和规格值，一个规格名称可能对应多个规格值。变体列表包含规格主图，价格，是否可购买，以及变体关联的规格组合列表。
   1. 每个商品都需要选择标签。
   2. 标签通过接口获取，获取后需要缓存到本地，点击刷新后强制调用接口更新本地的缓存。
   3. 标签是一个级联列表，标签支持多选，有MENU类型和其他类型，每个商品如果选择了一个MENU类型的叶子节点，就通过该节点的前缀和序列号拼装SPU（如"CUP0001）
   4. 选择的标签需要一目了然，支持直接点击进行删除。
2. 每个商品都有一个大图用于预览，而不是使用弹窗进行预览。点击商品列表的图片或者规格列表的规格图，就可以预览。
3. 第一个商品选择标签后，如果有多个商品，需要提示是否运用于其他标签。选择是之后，其他商品都需要选中这些标签。
4. 商品图片支持删除，规格选项也需要支持删除，规格选项删除后对应的变体需要同步删除。其中图片和规格选项删除前需要弹窗确认。
5. 商品支持批量导出为json，也支持调用接口批量同步到erp。导出或者同步成功后，提示用户是否清空商品列表。
6. 置顶的右侧工具栏有：导出、同步、刷新规格、清空商品列表、刷新商品列表。左侧显示插件名称。
7. 在独立标签页的头部展示一个面板能够查看正在获取商品的任务队列的进度百分比。还需要显示正在获取的商品总数，已获取的商品数量，正在获取的规格总数，已获取的规格总数。并结合脚本预测剩余时间。数据是通过定时器实现的。
8. 页面维护的定时器，用于获取任务队列的数据，每3秒执行一次，当本次调取和上一次调度获取到的任务队列不一致时，代表有新的任务进展，就是需要更新独立标签页头部展示的面板。分析任务队列更新的内容，如果是有新的商品数据获取完成，就从本地获取最新的商品列表，更新商品列表进行展示。

#### popup发起的任务队列描述

1. 任务队列支持热添加：当有进行中的任务时，能够继续获取和批量获取其他的商品。但是要求不能再次获取执行中的商品和已获取的商品。
2. popup能够通过的任务队列信息，定时计算并展示任务进度。要求展示总商品数量，已获取商品数量，总变体数量，已获取的变体数量。进度百分比=已获取的变体数量/总变体数量\*100%。
3. popup上可以点击取消任务，来清空任务队列，并终止正在进行的任务。如果终止失败，需要增加补偿方案（如：如果商品信息获取完成后，需要查看任务队列中是否有这个商品，如果有，才允许把这个商品保存到本地）

#### 商品信息获取

1. 获取商品信息后组装商品对象并保存到本地，数据结构和字段参考下方接口文档。注意平台的商品图片列表可能不包含规格主图，但是保存的数据里面，上面图片列表需要包含规格主图，注意数据去重。
2. 平台支持天猫、1688和淘宝，对应平台的数据结构可参考docs下的HTML文件。具体的脚本可以直接使用extractors目录下的js文件。如有需要可修改。

## 接口文档

### 1.标签级联列表接口

#### 接口基本信息

| 属性     | 值                                                     |
| -------- | ------------------------------------------------------ |
| 接口名称 | 获取标签级联列表                                       |
| 请求方式 | `GET`                                                  |
| 请求路径 | `/tag/treelist/ALL`                                    |
| 功能描述 | 获取系统中所有标签的级联列表结构，支持多级父子关系展示 |

#### 请求参数

该接口无需请求参数。

#### 请求示例

```http
GET http://127.0.0.1:8080/api/erp/tag/treelist/ALL
```

#### 响应参数

| 参数名 | 类型      | 必填 | 说明                     |
| ------ | --------- | ---- | ------------------------ |
| `code` | `integer` | 是   | 响应状态码，200 表示成功 |
| `msg`  | `string`  | 是   | 响应信息描述             |
| `data` | `array`   | 是   | 标签列表数据             |

**data 数组元素说明：**

| 参数名          | 类型      | 必填 | 说明                          |
| --------------- | --------- | ---- | ----------------------------- |
| `tagId`         | `integer` | 是   | 标签唯一标识                  |
| `tagName`       | `string`  | 是   | 标签名称                      |
| `tagCode`       | `string`  | 是   | 标签编码                      |
| `tagType`       | `string`  | 是   | 标签类型                      |
| `sortOrder`     | `integer` | 是   | 排序序号，数值越小越靠前      |
| `parentId`      | `integer` | 是   | 父级标签 ID，0 表示根节点     |
| `ancestors`     | `string`  | 是   | 祖先节点路径，格式如："0,1,2" |
| `menuLevel`     | `integer` | 是   | 菜单层级，1 表示一级          |
| `spuPrefix`     | `string`  | 是   | 商品SPU前缀                   |
| `currentMaxSeq` | `integer` | 是   | 当前最大序号                  |
| `children`      | `array`   | 是   | 子标签列表，递归结构          |

#### 响应示例

**成功响应：**

```json
{
  "code": 200,
  "msg": "success",
  "data": [
    {
      "tagId": 1,
      "tagName": "杯子",
      "tagCode": "CUP",
      "tagType": "MENU",
      "sortOrder": 1,
      "parentId": 0,
      "ancestors": "",
      "menuLevel": 1,
      "spuPrefix": "ACUP",
      "currentMaxSeq": 12,
      "children": []
    }
  ]
}
```

**响应字段补充说明：**

- `ancestors`：空字符串表示该标签为根节点，非空时格式为父级 ID 逗号分隔的路径
- `children`：子标签数组，结构与父级相同，支持无限层级嵌套
- `tagType`：枚举值，当前示例为 `MENU`，还有其他类型`OTHER`

### 2. 商品创建接口

#### 接口基本信息

| 属性     | 值                                                      |
| -------- | ------------------------------------------------------- |
| 接口名称 | 商品创建                                                |
| 请求方式 | `POST`                                                  |
| 请求路径 | `/product/selectionInfo`                                |
| 功能描述 | 根据商品名称、标签、SPU编码、规格选项及变体信息创建商品 |

#### 请求参数

| 参数名               | 类型             | 必填 | 说明                                                |
| -------------------- | ---------------- | ---- | --------------------------------------------------- |
| `productName`        | `string`         | 是   | 商品名称                                            |
| `tagIds`             | `array[integer]` | 否   | 标签 ID 数组，用于商品分类筛选                      |
| `sourceUrl`          | `string`         | 是   | 数据来源 URL 地址                                   |
| `spu`                | `string`         | 否   | 商品 SPU 编码，用于精确匹配商品                     |
| `mediaUrlList`       | `array[string]`  | 是   | 商品媒体文件 URL 列表，需包含规格主图（图片、视频） |
| `optionList`         | `array[object]`  | 是   | 商品规格选项列表                                    |
| `productVariantList` | `array[object]`  | 是   | 商品变体列表，定义不同规格组合的变体                |

**mediaUrlList 数组元素说明：**

| 参数名 | 类型     | 必填 | 说明                                 |
| ------ | -------- | ---- | ------------------------------------ |
| -      | `string` | 是   | 商品媒体文件 URL，支持多张图片或视频 |

**optionList 数组元素说明：**

| 参数名        | 类型            | 必填 | 说明                      |
| ------------- | --------------- | ---- | ------------------------- |
| `chineseName` | `string`        | 是   | 规格中文名称，如："颜色"  |
| `englishName` | `string`        | 否   | 规格英文标识，如："color" |
| `values`      | `array[object]` | 是   | 规格可选值列表            |

**values 数组元素说明：**

| 参数名         | 类型     | 必填 | 说明           |
| -------------- | -------- | ---- | -------------- |
| `chineseValue` | `string` | 是   | 规格值中文描述 |
| `englishValue` | `string` | 否   | 规格值英文标识 |

**productVariantList 数组元素说明：**

| 参数名              | 类型            | 必填 | 说明                                                   |
| ------------------- | --------------- | ---- | ------------------------------------------------------ |
| `purchasePrice`     | `number`        | 否   | 采购价格，精确到小数点后2位                            |
| `mediaUrl`          | `string`        | 否   | 变体对应的媒体文件 URL                                 |
| `isActiveAvailable` | `integer`       | 是   | 是否可购买，1 表示是，0 表示否。如获缺货或者禁用就为否 |
| `optionValueList`   | `array[object]` | 是   | 变体关联的规格值列表                                   |

**optionValueList 数组元素说明：**

| 参数名         | 类型     | 必填 | 说明                 |
| -------------- | -------- | ---- | -------------------- |
| `chineseValue` | `string` | 否   | 规格值中文描述，不传 |
| `englishValue` | `string` | 是   | 规格值英文标识       |
| `chineseName`  | `string` | 否   | 规格中文名称         |
| `englishName`  | `string` | 是   | 规格英文标识         |

## 4. 错误码说明

### 通用错误码

| 错误码 | 说明           | 处理建议                         |
| ------ | -------------- | -------------------------------- |
| `200`  | 请求成功       | -                                |
| `400`  | 请求参数错误   | 检查请求参数格式、必填项是否完整 |
| `401`  | 未授权访问     | 检查是否已登录或 Token 是否有效  |
| `403`  | 权限不足       | 检查用户权限配置                 |
| `404`  | 接口不存在     | 检查请求路径是否正确             |
| `500`  | 服务器内部错误 | 联系技术支持排查                 |

### 业务错误码

| 错误码 | 说明             | 处理建议                            |
| ------ | ---------------- | ----------------------------------- |
| `1001` | 标签不存在       | 检查 tagIds 是否有效                |
| `1002` | 商品不存在       | 检查 spu 编码是否正确               |
| `1003` | 规格选项无效     | 检查 optionList 格式和值是否合法    |
| `1004` | 变体信息不完整   | 检查 productVariantList 是否完整    |
| `1005` | 媒体链接失效     | 检查 mediaUrlList 中的 URL 是否有效 |
| `1006` | 商品名称不能为空 | 检查 productName 是否已填写         |
