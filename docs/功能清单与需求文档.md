# 📦 Velart Home 跨境电商 ERP 系统 PRD (v1.0)

## 1. 项目概述与架构

### 1.1 项目背景

本项目基于 **若依 (RuoYi)** 框架进行二次开发，旨在打造一个专属于 `Velart Home` 的现代化跨境电商 ERP 系统。核心解决从国内电商平台选品、本地化信息重构、智能定价、到 Shopify 多渠道分发及售后邮件监控的全链路闭环。

### 1.2 视觉与交互规范 (UI/UX)

- **设计语言**：采用现代化、极简的 B2B SaaS 风格（推荐参考 Arco Design 或 Element Plus 的现代主题）。
- **色彩规范**：主色调使用优雅的品牌色（如黑白灰辅以高级莫兰迪色），弱化边框，增加留白 (White Space)。
- **交互规范**：
  - 重度使用侧边抽屉 (Drawer) 和弹窗 (Modal) 以保持用户在当前工作流的沉浸感。
  - 图片展示采用瀑布流或整齐的网格布局，支持图片悬停放大 (Hover Zoom)。
  - 数据表格支持列自定义、冻结表头与平滑滚动。

### 1.3 技术栈约束 (提供给 AI 的开发指南)

- **前端**：Vue3 + TypeScript + Element Plus / Arco Design + Vuedraggable (用于拖拽排序) + 富文本编辑器 (WangEditor / TinyMCE)。
- **后端**：Spring Boot + MyBatis Plus + MySQL + Redis (若依微服务或单体架构)。
- **文件存储**：本地 NAS 挂载路径读写 (结合 Nginx 提供静态资源访问)。
- **外部集成**：Shopify Admin REST/GraphQL API、ExchangeRate-API、企业微信 Webhook、第三方 AI 去水印服务 (或本地 Python 脚本)、IMAP 邮件监听。

## 3. 核心业务模块需求

### 3.1 🏷️ 标签管理模块 (Tag Management)

_定义商品分类逻辑与营销属性的基石。_

- **标签分类**：
  - **菜单型标签 (Menu Tags)**：具备树形上下级结构（父子级）。核心字段：`标签名称`、`标签编码`、`父级ID`(0表示顶级菜单)、`SPU前缀` (英文字母缩写，如 `FURN001`)、`当前最大流水号`、。
  - **活动型标签 (Campaign Tags)**：扁平化结构，无上下级，仅用于商品系列管理使用。
- **核心逻辑**：生成 SPU 时，系统必须根据所选**菜单型标签的叶子节点** (Leaf Node) 提取其绑定的 `SPU前缀`，再结合`当前最大流水号`+1生成系统唯一的 SPU 码。

### 3.2 🛍️ 商品管理模块 (Product Management)

_系统核心：分为「选品抓取」与「详情维护」双流工作站。_

#### Step 1: 选品工作台 (Product Curation)

- **数据采集**：输入来源 URL (`sourceUrl`，如淘宝、1688、天猫、京东、拼多多)。系统需识别平台调用官方接口，自动解析并提取：`商品名称`、`主图列表`、`规格名称`、`规格值`、`规格图`、`不同规格的采购价 (CNY)`。
- **自动化入库**：
  - 点击保存时，后台异步将所有抓取到的图片下载并转存至 **服务器**。
  - 支持用户选择多个标签，但是菜单类型的标签必须是上下级关系，有且仅有一个叶子节点标签（关联 2.1 逻辑生成 SPU）。
- **规格重构**：
  - 支持对抓取到的中文规格（如 "颜色: 红色"）输入对应的英文翻译（如 "Color: Red"）。
  - **规格拓展**：支持手动增加货源平台没有的自定义规格名和规格值。
  - **规格排序**：系统自动对规格进行排序，并生成 `规格编码`。
  - **采购链接**：规格支持独立维护采购链接，默认为主商品(`SPU`)的采购链接。

#### Step 2: 详情维护工作台 (Detail Maintenance)

- **基础信息**：维护 `商品标题 (英)`、`productType`、`category`。商品描述支持 **富文本编辑器 (Rich Text)** 与 **HTML 源码** 一键切换。
- **图片绑定**：可视化浏览商品图片，支持拖拽快速绑定为特定规格的主图 (SKU Image)。
- **SKU 矩阵与智能计算 (核心逻辑)**：
  为每个 SKU 维护以下数据：包装尺寸 (长, 宽, 高)、常规实重、图片。系统需自动完成以下公式计算：
  - **材积重**：$$ Material Weight = \frac{Length \times Width \times Height}{DimFactor} $$ _(注: DimFactor需做成全局配置项，常默认为5000或6000)_
  - **实时汇率获取**：定时调用 `ExchangeRate-API` 获取实时 USD/CNY 汇率并缓存到redis，24小时有效。
  - **运费及成本**：根据包装尺寸 (长, 宽, 高)和常规包装重量调用4PX接口获取 `运费(CNY)`。$$ Cost(CNY) = 采购价(CNY) + 运费(CNY) $$
  - **智能定价模型**：设定目标利润率为 30%，系统自动推算建议售价：
    $$ Suggested Price (USD) = \frac{Cost(CNY)}{(1 - 0.30) \times Exchange Rate} $$
  - **实际利润计算**：用户手动输入 `销售价格(USD)`后，系统反算实际利润与利润率：
    $$ Profit (CNY) = (Sale Price (USD) \times Exchange Rate) - Cost(CNY) $$
        $$ profitRate (\%) = \frac{Profit (CNY)}{Sale Price (USD) \times Exchange Rate} \times 100\% $$

#### Step 3: Shopify 协同同步

- **动作**: 选中商品 -> 点击【批量推送】。
- **逻辑**:
  - 调用 Shopify GraphQL
  - **成功**: 更新本地状态为 `PUSHED`，记录 Shopify Product ID 、 shopify_variant_id 、shopify_url 和 shopify_image_id。
  - **失败**:
    - 捕获错误信息 (如: 标题过长, 价格格式错误)。
    - **通知**: 调用企业微信 Webhook，发送失败报告 (包含 SPU 列表和错误原因)。
    - **记录**: 本地数据库 `error_log` 字段记录详情。
    - **反馈**: 前端弹出摘要：“成功 X 个，失败 Y 个”，并提供“查看失败详情”链接。

### 3.3 📂 商品系列管理 (Collections)

- **创建规则**：与 Shopify 逻辑保持一致，基于 **标签 (Tags)** 自动/手动筛选聚合形成系列。
- **排序与陈列**：
  - 系列详情页中，以网格列表展示该系列下所有商品。
  - 支持 **拖拽排序 (Drag & Drop)** 调整商品展示先后顺序。
  - 提供 **「一键置顶」** 操作按钮。
- **同步机制**：支持将调整后的排序信息 (Sort Order) 同步更新至 Shopify 对应 Collection 的商品排序中。

### 3.4 🖼️ 媒体数字资产管理 (Media Assets)

- **存储目录结构**：基于 NAS，选品保存时自动以 SPU 编号创建专属文件夹。路径示例：`/nas/media/velart/{SPU}/`
- **命名与SEO规范**：
  - 规格主图：默认使用 `{SKU}.jpg` 命名。
  - 细节展示图：按 `{SPU}-0.jpg`, `{SPU}-1.jpg` 依次命名（支持前端重命名）。
  - **Alt Text 自动生成**：默认注入 `{{商品标题}} - Velart Home`。
- **AI 图像处理**：集成 AI 图像/视频处理能力（需定义对接接口），提供一键「去除水印」功能。
- **性能优化与压缩 (WebP/JPEG)**：
  - 系统后台支持大图原图展示，但不直接用于外部发布。
  - 触发同步 Shopify 前，系统自动执行图片压缩。压缩版本存放在对应 SPU 的子目录 `/press/` 下。Shopify API 上传仅读取 `/press/` 目录中的文件，节省带宽并提升独立站加载速度。

### 3.5 📧 邮件监控与消息推送 (Email & Webhook)

- **邮件监听服务**：后台常驻守护进程，基于 IMAP 协议实时监听指定的客服/业务邮箱（如 `support@velarthome.com`）。
- **企微触达**：解析收到的新邮件（发件人、主题、正文摘要），通过企业微信群机器人 Webhook，将通知实时推送到企微群。
  - _企微卡片示例_：

      "template_card": {

          { "keyname": "主题", "value": "{Subject}" }

      }
