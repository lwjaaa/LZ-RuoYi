# 表单快速提示词功能使用说明

## 功能概述

表单快速提示词功能用于记录用户在商品表单中常用的字段值，并在用户再次填写表单时提供智能建议，提高录入效率。

### 支持的字段

1. **基础字段**：
   - `size` - 商品大小
   - `material` - 材质
   - `note` - 备注（英文）
   - `noteCn` - 备注（中文）

2. **选项字段**（从 optionJson 中自动提取）：
   - `option_english_name` - 选项名称（如：Color, Size, Material）
   - `option_english_value` - 选项值（如：Red, Blue, M, L, Cotton）

## 核心特性

1. **智能排序**：按照最近使用时间排序，最新使用的值排在前面
2. **去重更新**：保存商品时，如果字段值未变化，则不更新使用时间
3. **批量查询**：一次性获取多个字段的提示词，减少网络请求
4. **自动维护**：保存商品时自动更新提示词使用记录

## 数据库表结构

### 表名：`erp_form_suggestion`

| 字段名 | 类型 | 说明 |
|--------|------|------|
| suggestion_id | bigint(20) | 主键ID，自增长 |
| field_name | varchar(50) | 字段名称（如：size, material, note, note_cn） |
| field_value | varchar(2048) | 字段值内容 |
| last_used_time | datetime | 最后使用时间 |
| create_time | datetime | 创建时间 |
| update_time | datetime | 更新时间 |
| del_flag | char(1) | 删除标志（0代表存在 1代表删除） |

### 索引设计

- **主键索引**：`suggestion_id`
- **唯一索引**：`uk_field_value` (field_name, field_value) - 防止重复
- **复合索引**：`idx_field_name_time` (field_name, last_used_time DESC) - 优化查询性能

## 后端接口

### 1. 批量获取提示词

**接口地址**：`POST /erp/formSuggestion/batchList`

**请求参数**：
```json
{
  "fields": [
    {"fieldName": "size", "limit": 30},
    {"fieldName": "material", "limit": 30},
    {"fieldName": "note", "limit": 30},
    {"fieldName": "noteCn", "limit": 30}
  ]
}
```

**返回结果**：
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": [
    {
      "suggestionId": 1,
      "fieldName": "size",
      "fieldValue": "M",
      "lastUsedTime": "2026-04-27 10:30:00"
    },
    ...
  ]
}
```

### 2. 批量更新提示词使用记录

**接口地址**：`POST /erp/formSuggestion/batchUpdateUsage`

**请求参数**：
```json
{
  "updates": [
    {"fieldName": "size", "oldValue": "M", "newValue": "L"},
    {"fieldName": "material", "oldValue": "棉", "newValue": "丝绸"}
  ]
}
```

**返回结果**：
```json
{
  "code": 200,
  "msg": "操作成功"
}
```

## 前端使用

### 1. API 调用

```javascript
import { batchGetSuggestions, batchUpdateUsage } from '@/api/erp/formSuggestion'

// 获取提示词
const fields = [
  { fieldName: 'size', limit: 30 },
  { fieldName: 'material', limit: 30 }
]
batchGetSuggestions(fields).then(response => {
  console.log(response.data)
})

// 更新使用记录
const updates = [
  { fieldName: 'size', oldValue: 'M', newValue: 'L' }
]
batchUpdateUsage(updates).then(response => {
  console.log('更新成功')
})
```

### 2. Element UI Autocomplete 集成

```vue
<template>
  <el-autocomplete
    v-model="form.size"
    :fetch-suggestions="querySizeSuggestions"
    placeholder="请输入大小"
    @select="handleSizeSelect"
    clearable
  ></el-autocomplete>
</template>

<script>
import { batchGetSuggestions } from '@/api/erp/formSuggestion'

export default {
  data() {
    return {
      suggestionsCache: {
        size: [],
        material: []
      }
    }
  },
  methods: {
    // 加载提示词
    loadSuggestions() {
      const fields = [
        { fieldName: 'size', limit: 30 },
        { fieldName: 'material', limit: 30 }
      ]
      
      batchGetSuggestions(fields).then(response => {
        if (response.code === 200 && response.data) {
          // 按字段名分组缓存
          const cache = { size: [], material: [] }
          
          response.data.forEach(item => {
            if (cache[item.fieldName]) {
              cache[item.fieldName].push({
                value: item.fieldValue
              })
            }
          })
          
          this.suggestionsCache = cache
        }
      })
    },
    
    // 查询建议
    querySizeSuggestions(queryString, cb) {
      const results = queryString
        ? this.suggestionsCache.size.filter(item => 
            item.value.toLowerCase().includes(queryString.toLowerCase()))
        : this.suggestionsCache.size
      cb(results)
    },
    
    // 选择建议后的回调
    handleSizeSelect(item) {
      console.log('选择了:', item.value)
    }
  }
}
</script>
```

## 业务流程

### 1. 打开商品编辑页面

1. 前端调用 `batchGetSuggestions` 接口获取提示词（包括基础字段和选项字段）
2. 将提示词按字段名分组缓存
3. 用户输入时，Autocomplete 组件从缓存中筛选匹配的建议

### 2. 保存商品

1. 前端提交商品数据到后端
2. 后端在 `ProductWizardServiceImpl.saveProductCommon` 方法中自动调用 `updateFormSuggestions`
3. 处理两种类型的字段：
   - **基础字段**：对比旧值和新值，如果未变化则不更新使用时间
   - **选项字段**：从 optionJson 中解析出所有的 englishName 和 englishValue，只添加新增的选项
4. 提示词更新失败不影响商品保存（异常捕获，仅记录日志）

## 注意事项

1. **性能优化**：
   - 使用复合索引优化查询性能
   - 前端缓存提示词数据，避免重复请求
   - 批量查询减少网络开销

2. **数据安全**：
   - 提示词更新失败不影响商品保存
   - 使用软删除机制（del_flag）
   - 唯一索引防止重复数据

3. **扩展性**：
   - 可以轻松添加新的字段支持
   - 只需在 `updateFormSuggestions` 方法的 fields 数组中添加字段名
   - 在前端添加对应的 Autocomplete 组件和查询方法

## 常见问题

### Q1: 如何添加新的字段支持？

**A**: 
1. 在后端 `ProductWizardServiceImpl.updateFormSuggestions` 方法的 fields 数组中添加字段名
2. 在 `getFieldValue` 方法中添加对应的 case 分支
3. 在前端添加对应的 Autocomplete 组件和查询方法
4. 如果是选项字段，需要在 `extractOptionSuggestions` 方法中添加解析逻辑

### Q2: optionJson 中的选项是如何提取的？

**A**: 
- 系统会自动解析商品的 optionJson 字段
- 提取所有的 `englishName`（选项名称）和 `englishValue`（选项值）
- 使用 Set 去重，避免重复添加
- 只添加新增的选项，已存在的选项不会重复更新
- 例如：optionJson 中有 `{"englishName": "Color", "values": [{"englishValue": "Red"}, {"englishValue": "Blue"}]}`
  - 会提取出：`option_english_name: "Color"`, `option_english_value: "Red"`, `option_english_value: "Blue"`

### Q3: 提示词数量限制是多少？

**A**: 默认每个字段返回 30 条记录，可以在调用 `batchGetSuggestions` 时通过 limit 参数自定义。

### Q4: 如何清理过期的提示词？

**A**: 可以通过定时任务定期清理长时间未使用的提示词，或者手动执行 SQL：
```sql
DELETE FROM erp_form_suggestion 
WHERE last_used_time < DATE_SUB(NOW(), INTERVAL 90 DAY);
```

### Q5: 提示词会影响商品保存性能吗？

**A**: 不会。提示词更新是异步的，且即使失败也不影响商品保存。整个更新过程在事务内完成，性能开销很小。
